カタログエントリの作成と維持方法。

## ファイル構成

```
catalog/
├── navigation/console-tree.yaml    # 1ファイル — 完全なメニューツリー
├── routes/<area>/<screen>.yaml      # コンソール画面ごとに1ファイル
├── resources/<resource>.yaml        # APIリソースごとに1ファイル
└── workflows/<resource>/<op>.yaml   # リソースごとの操作ごとに1ファイル
```

## 信頼度レベル

| レベル | 意味 | 使用時期 |
|-------|---------|-------------|
| `draft` | 未検証 | 初期作成、自動発見エントリ |
| `validated` | 確認済み | 指定バージョンのライブコンソールに対して検証済み |
| `stale` | 古い | 最後の検証以降にコンソールが変更された |

## 検証

`npm run validate` を実行して、すべてのカタログエントリをスキーマに対して確認します。
